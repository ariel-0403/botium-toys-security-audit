# Botium Toys Security Audit

This project contains a security audit conducted for a fictional company, Botium Toys, as part of my cybersecurity training.

## Objective

To assess the organization's security posture by reviewing existing controls, identifying risks, and evaluating compliance with industry standards such as PCI DSS, GDPR, and SOC.

## Key Findings

- Lack of access controls (no least privilege or separation of duties)
- No encryption for sensitive data
- No backups or disaster recovery plan
- Missing intrusion detection system (IDS)
- Weak password policies and no password management system

## Compliance Gaps

- Not compliant with PCI DSS requirements
- Partial compliance with GDPR
- Weak SOC controls

## Recommendations

- Implement least privilege and access control policies
- Deploy encryption for sensitive data
- Introduce IDS and monitoring systems
- Establish backups and disaster recovery plans
- Strengthen password policies and management systems
